Aggregate query in pymongo

Publish: 2019-03-20 Classify at: python
Words:1042 Read:≈ 5min

When using the mongo database, simple queries can basically meet most business scenarios, but just imagine, how to query if you want to count how many times a recommendation appears in the specified data? The stupid way is to use find to query the data, and then use the count() method for data statistics. This scenario is fine, but what if the sum of one of the fields is required? Is it necessary to traverse the corresponding data and then perform the sum operation? In mysql we often use count, group by and other queries, and in mongodb we can also use aggregation queries.

Suppose there is such a set of data

The price of each fruit is recorded in it. Now I want to count the number of times various fruits appear in this table. If there is no need to aggregate query, the idea should be like this, first take out all the data in the table, and then initialize a Dictionary, then traverse the data of each row, get its fName, and then update the count in the dictionary, the time complexity of this method is O(N), if the amount of data is large, it is not very good, the following Let’s take a look at how to query using aggregation.

The aggregate query uses the aggregate function, and its parameter is the pipeline pipeline. The concept of pipeline is to use the output result of the current command as the parameter of the next command. The pipeline is sequential, such as through the first pipeline. If there is no matching data after the operation, there will be no input for subsequent pipeline operations, so you must pay attention to the order of pipeline operations. Since for the above problem, we want all the data statistics, so there is no need for $match here

from pymongo import MongoClient

client = MongoClient(host=['%s:%s'%(mongoDBhost,mongoDBport)])
G_mongo = client[mongoDBname]['FruitPrice']

pipeline = [
        {'$group': {'_id': "$fName", 'count': {'$sum': 1}}},
    ]
for i in G_mongo['test'].aggregate(pipeline):
    print i

What if you want to get various stats with prices above 50? At this time, if there is a pipeline, the $match operation should be added before $group

1
2
3
4

pipeline = [
        {'$match':{'price':{'$gte':50}}},
        {'$group': {'_id': "$fName", 'count': {'$sum': 1}}},
    ]

Be sure to pay attention to the order

The conditions in $match are actually the same as using the find function.

Let’s focus on the $group operation. Group means grouping, which refers to which field the data is grouped according to. The above used {'$group': {'_id': "$fName", 'count': {' $sum': 1}, _id is the group to be divided, here is divided by the fName field, followed by 'count': {'$sum': 1}, here $sum It is the meaning of summation. The value behind is 1, that is to say, add 1 every time it occurs, so that the purpose of counting can be achieved. If you want to calculate the sum of price price, then it should be written like this

1

{'$group': {'_id': "$fName", 'count': {'$sum': '$price'}}}

Note that the fields here must have $, what if I want to ask for the average price? That is, first ask for the total number of prices, and then divide by the number of items, but there is a $avg operation

1
2
3
4

pipeline = [
        {'$match':{'price':{'$gte':50}}},
        {'$group': {'_id': "$fName", 'avg': {'$avg': '$price'}}},
    ]

There are many operations similar to $ave, the more commonly used ones are $min (to find the minimum value), $max (to find the maximum value)

pipeline = [
        {'$match':{'price':{'$gte':50}}},
        {'$group': {'_id': "$fName",
                    'count':{'$sum':1},
                    'priceAll':{'$sum':'$price'},
                    'avg': {'$avg': '$price'},
                    'min': {'$min':'$price'},
                    'max': {'$max':'$price'}
                    }
         },
    ]
for i in G_mongo['test'].aggregate(pipeline):
    print i

All supported operations can refer to the official documentation: group supported operations

_id must be used when grouping by which field.

Next, look at multi-key grouping. When using group for grouping query above, the _id used is a single field. For example, I have the following data in my database

With a user field, what if I want to group by user and fName? Here you can pass a dictionary into it

pipeline = [

        {'$group': {'_id': {'fName':'$fName','user':'$user'},
                    'count':{'$sum':1},
                    'priceAll':{'$sum':'$price'},
                    'avg': {'$avg': '$price'},
                    'min': {'$min':'$price'},
                    'max': {'$max':'$price'}
                    }
         },
    ]
for i in G_mongo['test2'].aggregate(pipeline):
    print i
