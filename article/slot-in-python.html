Usage of __slots__ in python

Publish: 2022-07-11 Classify at: python
Words:427 Read:≈ 3min

When looking at the sanic source code, I found that when defining a class, a __slots__ attribute will be set. Since I have not used a magic variable before, I checked it and record the function of this attribute here. We know that in a python instance, we can dynamically bind attributes and methods to the instance, like the following

class Test:
    def __init__(self, name, age):
        self.name = name
        self.age = age

t = Test("yangyanxing", 18)
print(t.name, t.age)
# Bind a city property to t
t.city = "BeiJing"
print(t.city)

We do not define a city attribute in the class, but we can add a city attribute to the Test instance, which is allowed in python. Of course I can also bind methods

from types import MethodType


class Test:
    def __init__(self, name, age):
        self.name = name
        self.age = age


def show(self, msg):
    print(f"hello {self.name}, {msg}")


t = Test("yangyanxing", 18)
# Bind the show method to t, here you need to use MethodType
t.show = MethodType(show, t)
t.show("Welcome")

But this binding is instance-specific, like the t instance above, if you instantiate another instance, it still doesn’t have a city property and a show method.

But sometimes, we don’t want to add attributes and methods to the instance randomly, or we want to limit the attributes and methods of the instance, we can only set the attributes and methods we specify, then we can use __slots__ to limit

from types import MethodType


class Test:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    __slotss__ = (
        "name",
        "age",
        "city",
        "show",
    )


def show(self, msg):
    print(f"hello {self.name}, {msg}")


t = Test("yangyanxing", 18)
print(t.name, t.age)
# Bind a city property to t
t.city = "BeiJing"
# Bind the show method to t
t.show = MethodType(show, t)
print(t.city)
t.show("Welcome")
t.school = "bjut"

The school here is not in __slots__, so it will crash here, don’t let it set. If you want to limit the attributes of the instance, you can set the __slots__ of the class.

However, subclasses are not restricted unless the subclass also sets the __slots__ attribute. If the subclass also sets the __slots__ attribute, then the attribute limit of the subclass is the sum of the parent class’s slots plus its own slots
