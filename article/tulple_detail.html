Don't underestimate tuples

Publish: 2018-11-28 Classify at: Python
Words:580 Read:≈ 3min

Tuples and lists, everyone who writes python knows and knows that tuples are immutable, but is it necessary to create a new data type just because one is immutable? The answer is definitely not, so let’s take a look at how tuples are used

1 Use tuples as records

Tuples are actually records of data: each element in the tuple holds the data for a field in the record, plus this The location of the field. It is this location information that gives meaning to the data For example, latitude and longitude information can be represented by a tuple

1
2
3

>>> lax_coordinates = (33.9425, -118.408056)
>>> print('%s/%s'%lax_coordinates)
33.9425/-118.408056

A tuple is used here to represent the latitude and longitude of a location, and its information can be printed directly during print.

2-tuple unpacking

  1. Assign values ​​separately
1

city, year, pop, chg, area = ('Tokyo', 2003, 32450, 0.66, 8014)

We assign the elements in the tuple (‘Tokyo’, 2003, 32450, 0.66, 8014) to the variables city, year, pop, chg and area, respectively, The most recognizable form of tuple unpacking is parallel assignment, that is, assigning the elements of an iterable object to a tuple of corresponding variables. 2. You can also use the * operator to split an iterable object as a function parameter

3 named tuples

This is rarely touched before, collections.namedtuple is a factory function, it can be used to build a tuple with field names and a named class, the memory consumption of the instance of the class constructed with namedtuple is related to the meta The groups are the same because the field names are stored in the corresponding classes. This instance is also smaller than a normal object instance.

#coding:utf-8
from collections import namedtuple

City = namedtuple('City', 'name country population coordinates')
Loca = namedtuple('Loca','lat,lon')

locatinfo = Loca(40.808611, -74.020386)

city ​​= City('New York-Newark', 'US', 20.104, locatinfo)

print city
print city.name
print city.country
print city.population
print city.coordinates
print city.coordinates.lat
print city.coordinates.lon

The print result is

City(name=‘New York-Newark’, country=‘US’, population=20.103999999999999, coordinates=Loca(lat=40.808610999999999, lon=-74.020386000000002)) New York-Newark US 20.104 Loca(lat=40.808610999999999, lon=-74.020386000000002) 40.808611 -74.020386

Creating a named tuple takes two arguments, one is the class name, and the other is the name of each field of the class.

Tuples are rarely used in daily work, mainly using lists more. Named tuples are very suitable for those relatively small classes, and can be used more in the future.
