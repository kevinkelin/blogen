Use gin to build redis of api background system

Publish: 2022-01-09 Classify at: Go
Words:948 Read:≈ 2min

Redis has a very important position in the web system and can act as a cache, message queue, step-by-step lock, etc. This article introduces the use of the go-redis library to simply operate redis.

Environment preparation

Start a redis locally using docker

docker run --name myredis -p 6379:6379 -d redis starts a passwordless redis server.

Install go-redis, go get github.com/go-redis/redis/v8

connect to redis

When I introduced MySQL before, a global MySQL object was defined in the project global. This method is still used here, and the Redis variable is declared in global/global.go

package db

import (
	"context"
	"database/sql"
	"fmt"
	"github.com/go-redis/redis/v8"
	_ "github.com/go-sql-driver/mysql"
	"time"
)



func InitDB() *sql.DB {
	dsn := fmt.Sprintf("%s:%s@(%s)/%s?charset=%s&parseTime=true&loc=Local",
		"root", "123456", "127.0.0.1:3306", "gintest", "utf8")

	if conn, err := sql.Open("mysql", dsn); err != nil {
		panic(err.Error())
	} else {
		conn.SetConnMaxLifetime(7 * time.Second) //Set the idle time, which is shorter than when mysql actively disconnects
		conn.SetMaxOpenConns(10)
		conn.SetMaxIdleConns(10)
		return conn
	}
}

func InitRedis() *redis.Client  {
	rdb := redis.NewClient(&redis.Options{
		Addr:	  "localhost:6379",
		Password: "", // no password set
		DB:		  0,  // use default DB
	})
	result := rdb.Ping(context.Background())
	fmt.Println("redis ping:", result.Val())
	if result.Val()!="PONG"{
		// 连接有问题
		return nil
	}
	return rdb
}

Personally, I prefer to use the configuration method, and I can see the various parameters intuitively.

If there is no problem with the connection, executing the Ping command will return a PONG string. If there is a problem, such as a problem with the connection of the redis command, an empty string will be returned. You can judge whether the redis server is normally connected based on this return value.

connection pool

In order to reduce frequent database connections, it is recommended to use a connection pool. Go-redis maintains available connections. When initializing a redis connection, use the PoolSize parameter to set the size of the connection pool.

	Addr: "localhost:6379",
	Password: "", // no password set
	DB: 0, // use default DB
	PoolSize: 10,
})

Operate redis

redis.Client has a lot of built-in operation methods, which can basically meet the general use, such as Get, Set, SetNX, etc. for operating strings, HSet for operating hash, etc. It is basically similar to redis-cli. The following ordinary strings are example

The return value of the SET command of redis is explained as follows

Starting from Redis 2.6.12, the SET command only returns OK if the set operation completed successfully; if the command uses the NX or XX options, but the set operation is not performed because the condition is not met , then the command will return a NULL Bulk Reply.

Reference: http://redisdoc.com/string/set.html

func (UserV1) RedisSet(c *gin.Context) {
	key := c.Query("key")
	val := c.Query("val")
	result, err := global.Redis.Set(c, key,val,0).Result()
	fmt.Println("err:", err)

}

In the V8 version of go-redis, the command to operate redis requires a context parameter, here you can pass *gin.Context, if there is no error in the above function, err is nil, and result is OK.

func (UserV1) RedisGet(c *gin.Context)  {
	key := c.Query("key")
	result := global.Redis.Get(c, key)
	res := make(map[string]string)
	val, err := result.Result()
	if err == redis.Nil{
		res["msg"] = fmt.Sprintf("没有key:%s ", key)
	}else if err!= nil{
		res["msg"] = err.Error()
	}else{
		res["msg"] = val
	}
	c.JSON(200, gin.H{"data": res})
}

There are various types of command execution results, some are String, some bool, and Scan type, depending on the source code. For example, Get returns *StringCmd, and its Result() method returns string, err, Another example is that SetNX returns the type *BoolCmd, and its Result() method returns bool, err, then you can judge directly.
