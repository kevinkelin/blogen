Using annotation functions in python

Publish: 2019-08-04 Classify at: Python
Words:340 Read:≈ 2min

Recently, I was looking at the source code of aiohttp and found a way of defining a function that I have never seen before. def __enter__(self) -> None, followed by a ->, which made a person who has been hurt by C pointers come to Said that the whole person is not good, so I checked the usage of this arrow with an arrow. The pointer of -> here has nothing to do with half a dime.

The arrow here is actually a kind of annotation, which is to tell the caller what the function returns, such as the following code

1
2

def test(x,y)->int:
    return x+y

This tells the user that this function will return a value of type int. Although the function annotation specifies that a value of type int is returned, we know that any type of value can be returned in the return value of python, and we can still return a value of type str without following the annotation.

1
2
3

def test(x,y)->int:
    r = x+y
    return "yangyanxing"

Although it will not report a syntax error, anyone who sees it will scold the street. Since you told your user that you want to return a value of type int, but you yourself return a value of type str to others, this is not true. compliant. When using the pyCharm editor, a warning is displayed

Expected type ‘int’, got ‘str’ instead

You can also use annotations for function parameters

Function parameters can also use annotations to tell the user what type of parameter to pass by adding a colon : after the parameter

1
2
3
4

def test(x:int,y:int)->int:
    return x+y

test(1,2)

The same is still the same problem, you can restrict, but I can also disobey, you ask me to pass an int, I prefer to pass a str, the syntax will not report an error, but a warning will be reported

Note that the above annotations are written only in python3. If you run it under python2, a syntax error will be reported!
