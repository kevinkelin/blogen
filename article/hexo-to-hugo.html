hexo migrate hugo

Publish: 2021-05-31 Classify at: web
Words:1609 Read:≈ 8min

Finally I can’t stand hexo anymore. I changed the computer. Basically, the original things can’t be migrated. The node version, the dependencies of various plug-ins, and different versions will be different. I encounter various compilation errors and the compilation is slow. . . .

I couldn’t take it anymore, and finally decided to migrate my blog from hexo to hugo. I was a little hesitant at first, but after I turned it over, it was really fragrant!

The following problems were encountered during the migration process

  1. Keep the url of the original blog
  2. The md file format of the original blog is incompatible with some formats of hugo
  3. Customize the content of the sidebar to display the WeChat public account
  4. Add comment component
  5. Modify the search
  6. Add archive page, tag page, category page

1. Keep the url of the original blog

This is still more important, because the previous blog address has been indexed on the search engine, the previous url is in the form of /article/xxxx.html, but the default format of hugo is /post/xxxx, If it is re-indexed, it will take a long time to be indexed by search engines. In order to keep the same as before, when creating a new article here, you need to use the following command

1

hugo new article/xxxx.md

This command will create a new article md file to the content/article directory, but the generated article path is /article/xxxx, which has a .html suffix from the previous article, and will generate a directory for each article.

The solution is to modify the site configuration file config.toml, add

1

uglyURLs = true

After adding this configuration, the generated article url will be suffixed with .html. I don’t know why this url is ugly. Anyway, I don’t think it is ugly. . .

When the above format is converted, first there is no --- at the top, without these three bars, hugo will not consider the following content as frontmatter information, but treat it as the text of the article, which will cause the article to have no title, no Tags, you need to add --- at the top of all articles.

But this logo has a space before and after the more word, hugo does not recognize it, the solution is to remove the spaces before and after more.

Before md used permalink to customize the url of the article, in hugo, you need to change permalink to slug

3. Customize the content of the sidebar to display the WeChat public account

The previous blog used the Maupassant theme, mainly because it is very concise. This theme also has a corresponding theme on hugo, [maupassant-hugo](https:// github.com/flysnow-org/maupassant-hugo) , in fact, this theme already meets my needs very well, but because the articles in my blog are all written in the article directory, the traversal articles of this theme do not contain this theme. directory, so you need to modify the theme template.

All articles under the search topic contain slice "post" "posts" , this code means to traverse the files in the post and posts directories, modify it to slice "post" "posts" "article", add An "article" directory so that my custom directory is traversed.

There is an advertisement on the right sidebar of this theme by default. I can’t use it now, so I deleted it, but what I need is to put my WeChat official account on the right sidebar, so according to the writing method of its advertisement template, I added an advertisement for To display the template of WeChat official account, add the mp.html file in the layouts/partials directory of the theme, and write the following code

Fourth, add a comment component

This is relatively rough, there is no perfect solution at present, I have tried Gitalk, gitment, utterances.

They are all based on github issues to submit comments, but Gitalk needs me to take the initiative to visit the article, so many need to be revisited before,

utterances does not need to be installed. His application needs to be installed on github, but when I deployed it, there was still a problem with his service. . . After submitting a comment, it shows 403. I have been looking for whether there is something wrong with my configuration, until I see that everyone has submitted issues on its github. . .

Five, modify the search

The default search of this theme is to use google, but as you all know in the domestic environment, I wanted to change to Baidu site search, but I found that the time for Baidu search to be included is too slow, this is still when I added a sitemap to Baidu , and sometimes it can’t be included in a month, but I found that bing’s index is very fast, usually it can be climbed and included in one day, and I haven’t submitted a sitemap at all, I even doubt whether bing uses google’s retrieval. . . .

Bing search is also highly recommended here. Recently, it has been found that it is basically similar to google in terms of searching for technical articles, and there are fewer advertisements.

Change the search engine to bing.

But this is not enough. After submitting the search, the site search will not be performed. You need to write a js file to add site:www.yangyanxing.com into it.

$(window).scroll(function() {
    $(window).scrollTop() > 500 ? $("#rocket").addClass("show") : $("#rocket").removeClass("show");
});
$("#rocket").click(function() {
    $("#rocket").addClass("launch");
    $("html, body").animate({
        scrollTop: 0
    }, 500, function() {
        $("#rocket").removeClass("show launch");
    });
    return false;
});

// bing search

function bingso(){
    var querystr = $("input[name='q']")
    var searchform = $("#search")
    var q = querystr.val()
    q = q + "site:www.yangyanxing.com"
    querystr.val(q)
    searchform.submit()
    querystr.val("")
}

$('#search').bind('keydown', function(event) {
    if (event.keyCode == "13") {
        // Enter to execute the query
        bingso()
        return false
    }
});

The bing search above is the original content of the theme, which is the button to return to the top in the lower right corner of the web page. The content below is to add site:www.yangyanxing.com to the original search term when searching, and then continue the search .

After a lot of tossing, I finally completed the conversion of the blog from hexo to hugo. There is not much change on the interface, but for the various node_modules of hexo that have been tormented before, it is now too refreshing. All the articles in the blog Upload it to github, and on any computer, you can write as long as you git pull it down,and finally you don't have to be tortured by node anymore.
