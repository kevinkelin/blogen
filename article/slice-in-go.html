slice as function parameter in golang

Publish: 2021-10-15 Classify at: Go
Words:709 Read:≈ 4min

The slice in golang is very similar to the list in python, but there are many differences. This article summarizes some features when the parameter of a function is a slice type.

Arrays as arguments to functions

package main

import (
"fmt"
)

func main() {
	a := [2]int{1,2}
	fmt.Println(a)
	arraytest(a)
	fmt.Println(a)	
}


// parameter is an array
func arraytest(a[2]int) {
// try to modify the value in the array
	a[0] = a[0]*2
}

The above code prints the result

[1 2]
[1 2]

It means that when an array is passed into the function as a parameter, modifying the array in the function body will not affect the original array, because in the function body, the actual operation is the copy of the parameter

If you want to modify the array within the function and also change the original array, the parameter should be changed to a pointer type

package main

import (
"fmt"
)

func main() {
	a := [2]int{1,2}
	fmt.Println(a)
	arraytest(&a) //The address of array a is to be passed in here
	fmt.Println(a)
}


// parameter is an array pointer
func arraytest(a *[2]int) {
// try to modify the value in the array
	a[0] = a[0]*2
}

But it should be noted here that all functions are copies of operation parameters, and the above functions are actually copies of operation a

package main

func main() {
	a := [2]int{1,2}
	fmt.Println(a) // output is [1,2]
	var s_a = a[:]
	slicetest(s_a)
	fmt.Println(a) // output is [2,2]
}


// parameter is slice
func slicetest(a[]int) {
	a[0] = a[0]*2
}

Modify the slice in the function, which will affect the original array in this case!

	sam := make([]int, 2)
	sam = []int{1,2}

	fmt.Println(sam)

// parameter is slice
func slicetest(a[]int) {
	a[0] = a[0]*2
}

In this case, the modification of parameters will also affect the original slice.

  1. Slice pointer

In the go language, the slice name itself is an address. Accessing slice elements by adding a subscript to the slice name is originally a manifestation of pointer access. It is best not to use the pointer again in the function parameters. trouble

package main

import (
	"fmt"
)

func main() {
	sam := make([]int, 2)
	sam = []int{1,2}
	fmt.Println(sam)
	slicetest(&sam) //The address of sam is to be passed here
	fmt.Println(sam)
}


// parameter is slice
func slicetest(a *[]int) {
  // Be sure to enclose *a here, *a represents the value at a, which is the slice itself
  (*a)[0] = 100
}

package main

import (
"fmt"
)

func main() {
	// initialize the slice with new
	san := new([]int)
	san = &[]int{1,2}
	slicetest(san)
	fmt.Println(*san)
}


// parameter is slice
func slicetest(a *[]int) {
  // Be sure to enclose *a here, *a represents the value at a, which is the slice itself
  (*a)[0] = 100
}

Using new to create a variable returns the address of the variable, so the type of san in the above code is *[]int, which is itself a pointer, so it can be directly passed into the slicetest function, san := new([]int ) just declares the variable, but it is not initialized. It is an empty slice. If it is passed into the function at this time, the index overflow will be caused by the line (*a)[0] = 100.
