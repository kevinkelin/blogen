Use Uiautomator for UI-based testing

Publish: 2016-05-12 Classify at: Android
Words:856 Read:≈ 5min

I have been watching the test of the native uiautomator in the past two days, and I have used it before, but I just implemented the official Android example simply, and there is no actual test meaning. [Implementation of the official Android example of uiautomator](http:/ /www.yangyanxing.com/article/1264.html), today combined with the automation project I am doing recently, I simply used uiautomator to implement it, just made a small example, but basically how to compile the jar and run it Well, here is also a simple record. I have also used the python packaged by others to do it before, and that one is relatively simple to write. [Using uiautomator’s python package for android UI testing]( http://www.yangyanxing.com/article/how-to-use-uiautomator -with-python.html)

Create a new java project

When creating a new project, pay attention to the encoding of utf-8, otherwise there will be problems when ant is used to compile the jar file later. Introduce JUnit and android.jar and uiautomator.jar, I use the jar of api-16, remember this 16, it is useful to generate build.xml later

create test class

The new class should inherit the UiAutomatorTestCase class I use the am command for this operation to start the mobile phone first aid kit and close the first aid kit, so I created a util class to seal two methods

package com.yangyanxing.test;

import java.io.BufferedReader;
import java.io.InputStreamReader;

import com.android.uiautomator.core.UiObject;
import com.android.uiautomator.core.UiSelector;

public class TestUtil {
    
    public static String doCmdshell(String commond){
        String s = null;
        try
        {
            Process p = Runtime.getRuntime().exec(commond);
            BufferedReader stdInput = new BufferedReader(new InputStreamReader(p.getInputStream()));
            BufferedReader stdError = new BufferedReader(new InputStreamReader(p.getErrorStream()));

            String result = "";
            while ((s = stdInput.readLine()) != null)
            {
                result = result + s + "\n";
            }
            while ((s = stdError.readLine()) != null)
            {
                System.out.println(s);
            }
            return result;
        }
        catch (Exception e)
        {
            return "Exception occurred";
        }
    }
    
    //This method is to wait for the appearance of the ui element within the timeout time, return true if it occurs, return false if it is abnormal or timed out
    public static Boolean waitForUiselectorAppears(UiSelector selector,int timeout)
    {
        UiObject uiObject = new UiObject(selector);
        return uiObject.waitForExists(timeout*1000);
    }
    

}

package com.yangyanxing.test;

import com.android.uiautomator.core.UiDevice;
import com.android.uiautomator.core.UiObject;
import com.android.uiautomator.core.UiObjectNotFoundException;
import com.android.uiautomator.core.UiSelector;
import com.android.uiautomator.testrunner.UiAutomatorTestCase;

public class UitestRunner extends UiAutomatorTestCase {
    //Initialize a UiDevice
    private UiDevice mDevice = UiDevice.getInstance();
    
    public UitestRunner(){
        super();
    }
    
     
    
    //Write the setUp() method
    public void setUp() throws Exception{
        super.setUp();
        //Every time you test, you need to start the first aid kit, so put this method in setUp
        TestUtil.doCmdshell("am start com.qihoo.mkiller/com.qihoo.mkiller.ui.index.AppEnterActivity");
    }
    
    //Write the tearDown method to force-stop the first aid kit
    public void tearDown() throws Exception{
        super.tearDown();
        TestUtil.doCmdshell("am force-stop com.qihoo.mkiller");
    }
    
    //Detect if there is a "Start Scan" button after the first aid kit is activated

    public void test_startScanButton() throws UiObjectNotFoundException{
        UiSelector scanButton = new UiSelector().className("android.widget.Button").text("Start Scan");
        if(TestUtil.waitForUiselectorAppears(scanButton, 20)){
            UiObject scanoObject = new UiObject(scanButton);
            if (scanoObject.click()) {
                System.out.println("Start Scan button was clicked!");
            }else{
                System.out.println("Start Scan button click failed");
            }
        }else {
            System.out.println("First aid kit failed to start");
        }
        
        UiSelector exitbutton = new UiSelector().className("android.widget.Button").text("exit");
        
        assertEquals(Boolean.TRUE, TestUtil.waitForUiselectorAppears(exitbutton, 120));
    }
}

Use android.bat to generate the build.xml file

Go to the tools directory of the android sdk directory and use

android create uitest-project -n mkillerTest -t 2 -p D:\EclipseWorkPlace\uiautomatorTest`

command to generate build.xml

-n is the name of the final generated jar package, you can start it at will -t is the id of android targets, how to get it later -p is the project directory just created

The value of -t can be obtained through the android list in the sdk/tools directory

The api version of uiautomator just used is 16, then the corresponding id value here is 2

Use ant build… to generate mkillerTest.jar file

Refresh the project directory in eclipse, the build.xml will come out, Right-click build.xml and select Run as->Ant build…. Check build[from imort${sdk.dir}/tools/ant/uibuild.xml] in the Targets tab Then click run, if there is no error, it will eventually generate a BUILD SUCCESSFUL prompt and print the jar file location

Push the generated jar to the phone for testing

Use the adb push command to push the mkillerTest.jar to the /data/local/tmp/ directory of the phone

adb push mkillerTest.jar /data/local/tmp/

run tests

adb shell to the mobile phone, cd to the /data/local/tmp directory, or not to this directory (as long as the .jar file is pushed to the /data/local/tmp directory before), run the uiautomator command to test

uiautomator runtest mkillerTest.jar

or

uiautomator runtest mkillerTest.jar -c com.yangyanxing.test.UitestRunner

The -c command runs for the specified class. When there are multiple classes, you can choose to run only some use cases
