Recently, I am sorting out some queries that are relatively rarely used in pymongo.

ignore case query

You can use regular (re) to query

1
2
3
4
5
6
7

import re
import pymongo

name = r"^%s$"%filename
re_name = re.compile(name, re.I)
conn = pymongo.MongoClient(host, port)
fileinfos = conn.fileRecode.fileTask.find({'filename':re_name},projection={'_id': False})

You can also use $regex expressions, the above code can be replaced with the following

1

fileinfos = conn.fileRecode.fileTask.find({'filename':{'$regex':r"^%s$"%filename,'$options':"i"}},projection={'_id': False })

$options currently supports four kinds, the more commonly used is i

Options Meaning Usage Requirements
i Case insensitive
m Anchors are used in query matching, such as ^ (representing the beginning) and $ (representing the end), and matching strings after \n
x Ignore all whitespace Requires $regex to be used with $option
s allows the dot character (.) to match all characters, including newlines. Requires $regex to be used with $option

1

Query data for the specified time period

Sometimes when inserting data, datetime.datetime.now() is used to insert data in datatime format. For example, how to query the data of a certain period of time? You can use $gte (greater than or equal to) and $lte (less than or equal to) to query, for example, to query the data of one year, you can do this

1
2
3
4
5
6
7

import date
from pymongo import Connection

start_check = datetime.datetime.now()-datetime.timedelta(days=365)
checkCon = {'addtime':{"$gte":start_check}}
conn = Connection(host, port)
fileinfos = conn.Documents.find(checkCon)

#Logical relationship query Common logical queries include and or not

  1. Relationship and (and) In fact, this does not require special keywords, and directly writes the content to be queried into the query conditions in turn, which is the relationship between and
1
2

>>> conn.yang.test.find_one({"info.name":"fanjy","id":1})
{u'info': {u'age': u'20', u'name': u'fanjy'}, u'_id': ObjectId('5c24a2acdb1f5a209cf8799b'), u'id': 1}
  1. The relationship or (or) needs to use $or, for example, to query the name of yangyanxing or the address in tongzhou
1

coon.yang.test.find({"$or":[{"name":"yangyanxing"},{"address":"tongzhou"}]})

The value of $or is a list, each item in the list is a dictionary, and the dictionary is various query conditions.

Lists can also be iterated over lists. For example, if I want to find tasks with taskids 1, 2, 4, 6, 7, 9, 10, 20, and 34, I can use

1

conn.yang.test.find({'$or':[{"taskid":i} for i in l]})
