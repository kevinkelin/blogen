List sorting problems in python and the use of sort and sorted

Publish: 2018-06-28 Classify at: Python
Words:620 Read:≈ 3min

List sorting is a very common operation. There are many methods and pits. Here is a simple record.

  1. The sorted method

>>> l = ['1','2','A','z','a',1,2]
>>> sorted(l)
[1, 2, '1', '2', 'A', 'a', 'z']
>>> l
['1', '2', 'A', 'z', 'a', 1, 2]
>>> l2 = sorted(l)
>>> l2
[1, 2, '1', '2', 'A', 'a', 'z']
>>> id(l2)
38967136
>>> id(l)
38985520

The sorted method passes in a list and returns a new list, which does not affect the order of the original list. From the above results, the numbers are sorted before the strings. In fact, they are sorted according to the ASCII order of the characters. If What about strings in the list?

1
2
3

>>> l = ["yang","yan","xing","is","a","boy"]
>>> sorted(l)
['a', 'boy', 'is', 'xing', 'yan', 'yang']

Obviously it is arranged according to the first character, the first character is the same and then the second row is arranged.

The sorted() method has another parameter, reverse, which, when True, will sort in reverse order.

1
2

>>> sorted(l,reverse=True)
>>> ['yang', 'yan', 'xing', 'is', 'boy', 'a']

Is it possible to directly change the structure of the original list instead of generating a new list? sure

  1. sort method
>>> l
['yang', 'yan', 'xing', 'is', 'a', 'boy']
>>> l.sort()
>>> l
['a', 'boy', 'is', 'xing', 'yan', 'yang']
>>>

At this time, the structure of the original list l has changed, and the function has no return value, so if you write newL = l.sorr(), you can’t get a new list, newL is empty, so don’t use it wrong

This sort is too simple, can I sort it according to my own method? sure

  1. Custom sorting

A sorting method needs to be provided. This method has three return values. If there are two values ​​a1 and a2, through some calculation, if a value greater than 0 (True) is returned, then a1 is ranked behind, if it is less than 0 (False) ), a1 is in the front, if it is equal to 0, the order of a1 and a2 remains unchanged, see the code

a = [
    {'name':'jk', 'score':100, 'first':50, 'second':50, 'third':0},
    {'name':'zz', 'score':90, 'first':50, 'second':30, 'third': 10},
    {'name': 'yyx', 'score':90, 'first':40, 'second':30, 'third':20},
    {'name': 'zs', 'score':90, 'first':40, 'second':30, 'third':20},
]

def cmp(a1, a2):
    if a1['score'] != a2['score']:
        return a1['score'] - a2['score']
    elif a1['first'] != a2['first']:
        return a1['first'] - a2['first']
    elif a1['second'] != a2['second']:
        return a1['second'] - a2['second']
    else:
        return a1['third'] - a2['third']
        
a.sort(cmp)

for i in a:
    print i

This leads to another problem. Dictionaries are unordered. If you want to arrange dictionaries, you are actually arranging a group of dictionaries. Here I hope to sort the above list according to the ‘score’ in each record. If the values ​​of the ‘score’ fields are equal, sort by the value of ‘first’. If ‘first’ is still equal, sort by ‘second’. If ‘second’ is equal, sort by the value of the ’third’ field. The result of this sort is

{‘second’: 50, ‘score’: 100, ’name’: ‘jk’, ’third’: 0, ‘first’: 50} {‘second’: 30, ‘score’: 90, ’name’: ‘zz’, ’third’: 10, ‘first’: 50} {‘second’: 30, ‘score’: 90, ’name’: ‘yyx’, ’third’: 20, ‘first’: 40} {‘second’: 30, ‘score’: 90, ’name’: ‘zs’, ’third’: 20, ‘first’: 40}
