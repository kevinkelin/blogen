Receiver type problem of structure method in golang

Publish: 2021-10-17 Classify at: Go
Words:869 Read:≈ 5min

The structure in golang can be regarded as a class in object-oriented programming. You can define methods for the structure. Note the difference between methods and functions here. The definition of a function has no receiver, and the method has a receiver. Here The receiver can be in instance pointer form or instance form, for performance reasons recv is most commonly a pointer to receiver_type, (because we don’t want a copy of the instance, as it would be if call-by-value) , especially when the receiver type is a struct.

method definition

package main

import "fmt"

type Person struct {
	name string
	age int
}

func (self Person) sayname() {
	fmt.Printf("My name is %s and age is %d \n", self.name, self.age)
}

func main() {
	p := Person{
	"yangyanxing", 18,
	}
	p.sayname()
}

The result of running here is My name is yangyanxing and age is 18 , I use func (self Person) sayname() when I define the sayname method. The difference here is that the function name is preceded by a A receiver (self Person), here I use self, since self is not a keyword in go, I use the keyword of class in python, of course, any valid variable can be used here, if Before writing java, you can also use this, which is essentially equivalent to the instance itself.

The method defined here does not use the pointer method, so the method body is a copy of the variable to be operated. If the structure is relatively large, or even if it is not large, it will have some impact on systems with relatively high performance requirements, so in general , the method is defined in the form of a pointer to a structure.

func (self *Person) sayname() {
	fmt.Printf("My name is %s and age is %d \n", self.name, self.age)
}

	pt.sayname()

Here, both p and pt can call the sayname method normally, pointer methods and value methods can be called on pointers or non-pointers, which will be automatically converted inside golang.

The above sayname method does not modify the value of the variable itself. If you need to modify the value of the variable, you need to use a pointer.

String() method

Everyone who uses java should know the toString () method, and everyone who uses python should also know the __str__ method. These methods are used to print the object itself. When the class print function is called, the string returned by the method will be printed. Structures in golang also have a String() method for printing the structure

package main

import "fmt"

type Person struct {
	name string
	age int
}

func (self Person) String() string {
	return fmt.Sprintf("Person name is %s and age is %d", self.name, self.age)
}

func main() {
	p := Person{
	"yangyanxing", 18,
	}
	fmt.Println(p)
}

This prints Person name is yangyanxing and age is 18

However, it should be noted here that if the value form is used when the method is defined, then the value form must also be used when the method is called. If the pointer form is used when the method is defined, the structure address must also be used when calling.

package main

import "fmt"

type Person struct {
	name string
	age int
}

func (self *Person) String() string {
	return fmt.Sprintf("Person name is %s and age is %d", self.name, self.age)
}

func main() {
	p := Person{"yangyanxing", 18}
	pt := &Person{"yyx", 20}
	fmt.Println(p) //{yangyanxing 18}
	fmt.Println(&p) //Person name is yangyanxing and age is 18
	fmt.Println(pt) //Person name is yyx and age is 20
	fmt.Println(*pt) //{yyx 20}

}

Summarize

  1. For performance reasons, the structure method receiver is generally in the form of a pointer
  2. Both pointer methods and value methods can be called on pointers or non-pointers
  3. The String() method needs to pay attention to the receiver type. When using the print method, the parameters must be the same as the receiver type of the structure method.
