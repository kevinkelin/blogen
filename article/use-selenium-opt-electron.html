Use selenium to operate electron applications

Publish: 2021-10-14 Classify at: python
Words:1432 Read:≈ 7min

In a recent automated test project, there is a requirement for testing internal IM communication software. After understanding the implementation of the corresponding technology stack of each client, the application in the mac uses the electron technology. We also conduct research on the corresponding technology. The article records the automated testing of electron applications.

1. Introduction to electron

Electron uses Chromium and Node.js to build cross-platform desktop applications using JavaScript, HTML and CSS. It can compile the code at one time and run multiple terminals. It can generate mac app, windows exe, etc., but the compiled files are due to Contains Chromium, so the volume is generally larger.

Second, use selenium to operate the Chrome browser

First determine the version of the chrome browser in the system. Since selenium needs to use the ChromeDriver to operate the Chrome browser, different versions of Chrome correspond to different ChromeDrivers. If they do not correspond, it may fail.

Then go to https://chromedriver.chromium.org/downloads to download the corresponding ChromeDriver. If domestic users cannot access it, they can use the Taobao mirror source https://npm.taobao.org/mirrors/chromedriver

You need to put chromedriver in the environment variable. If you don’t put it in the environment variable, you can also specify the path of chromedriver in the code.

assert "No results found." not in driver.page_source
driver.close()

Third, use selenium to operate electron applications

If you want to use selenium to operate an electron app, the first thing to confirm is which version of Chromium the app is using

So how to confirm the Chromium version number of the electron application?

In general, the apps we test are all apps packaged by development. For this kind of application, it is difficult for us to do some hooks on the code. If we open the chrome devTools panel, (use option+command+i under mac) , Enter process.versions in the console tab to view the chrome version, then it’s better, you can directly download the corresponding chromedriver, but more often the process object is not defined, if you want to use the console Using process, when writing an app, you should add

const { app, BrowserWindow } = require('electron')
const path = require('path')

function createWindow () {
  const win = new BrowserWindow({
    width: 800,
    height: 600,
    webPreferences: {
      preload: path.join(__dirname, 'preload.js')
    }
  })

  win.loadFile('index.html')
}

app.whenReady().then(() => {
  createWindow()

  app.on('activate', () => {
    if (BrowserWindow.getAllWindows().length === 0) {
      createWindow()
    }
  })
})

app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') {
    app.quit()
  }
})

from selenium import webdriver
from selenium.webdriver.common.keys import Keys
import time


options = webdriver.ChromeOptions()
options.binary_location = "/Applications/electroltest.app/Contents/MacOS/electroltest"
driver = webdriver.Chrome(chrome_options=options)
time.sleep(3)
print(driver.page_source)
driver.get("http://www.google.com")
elem = driver.find_element_by_name("q")
elem.clear()
elem.send_keys("360") # Chinese can also be entered here
elem.send_keys(Keys.RETURN)
driver.close()
driver.quit()

The more important thing here is to define options.binary_location which needs to be set as the path of the electron application

Fourth, use selenium to operate the opened electron application

In the previous way, we actually reopened an app, but more often, we expect to test an app that has already been opened. Imagine the following scenario, an app needs to be logged in, and the login method is more complicated. You can log in with code or SMS, or have a verification code, but if you log in successfully, you will not enter the login interface if you open it again after a period of time. At this time, we do not want to log in again every time you run the use case. We need to use selenium to test an open application.

An app built with electron can add a port for debugging at startup

1

/Applications/electroltest.app/Contents/MacOS/electroltest --remote-debugging-port=9222

In this way, while the application can be started normally, the remote debugging function will also be enabled. When we use selenium, we can directly test the app without having to reopen a new app again.

The port here needs an unoccupied port, which can be used when initializing the driver.

1
2
3

options = webdriver.ChromeOptions()
options.add_experimental_option("debuggerAddress", "127.0.0.1:9222") # Set debuggerAddress
driver = webdriver.Chrome(chrome_options=options)

After passing the above settings, we can test directly on the open application, which is very suitable for applications with complicated login.

5. Summary

There are a few points to summarize the tests of electron applications using selenium

  1. Find the corresponding chromedriver
  2. Set the app path through the binary_location parameter
  3. For testing an open app, you need to add the remote debugging parameter --remote-debugging-port when the app starts
