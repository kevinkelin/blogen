Cross-domain problem of using gin to build api backend system

Publish: 2022-01-14 Classify at: Go
Words:702 Read:≈ 2min

In the development mode of front-end and back-end separation, the cross-domain problem has always been a common problem. The general solution is divided into front-end and back-end. The front-end is mainly solved by jsonp, but the back-end solution will be more convenient. This article records the use of gin The framework solves cross-domain problems in the backend.

Prepare frontend files

Prepare a file for sending http requests on the front end. This file uses jquery to send ajax, of course, you can also use vue+axios

For the sake of demonstration, I simply use python to start an http service

python -m SimpleHTTPServer 8888

At this time, if this file is opened in the browser, or nginx is used as a static proxy, as long as the file is not a request initiated through the http://127.0.0.1:8080 domain, due to the browser’s same Origin policies are blocked.

Access to XMLHttpRequest at ‘http://127.0.0.1:8080/api/v1/test’ from origin ‘http://127.0.0.1:8888’ has been blocked by CORS policy: No ‘Access-Control-Allow- Origin’ header is present on the requested resource.

##gin backend to solve cross-domain

When the browser sends an http request, it will record the Origin information in the header

Let’s first look at how gin obtains the request header information. gin obtains the request header information through *gin.Context.Request.Header

func (ApiV1) Get(c *gin.Context) {
	fmt.Println(c.Request.Header)
	c.JSON(200, gin.H{"msg": "handlers get request!"})
}

To get the Origin information, the Header itself is a map[string][]string type data, and the Origin information may be obtained through the Get method

Add this field to Access-Control-Allow-Origin in the response header and also set Access-Control-Allow-Methods

func (ApiV1) Get(c *gin.Context)  {
	origin := c.Request.Header.Get("Origin") 
	if origin!=""{
		c.Header("Access-Control-Allow-Origin", origin)  // 
		c.Header("Access-Control-Allow-Methods", "POST, GET, OPTIONS, PUT, DELETE, UPDATE")
		c.Header("Access-Control-Allow-Headers", "Origin, X-Requested-With, Content-Type, Accept, Authorization")
		c.Header("Access-Control-Expose-Headers", "Content-Length, Access-Control-Allow-Origin, Access-Control-Allow-Headers, Cache-Control, Content-Language, Content-Type")
		c.Header("Access-Control-Allow-Credentials", "true")
	}
	c.JSON(200, gin.H{"msg": "handlers get request!"})
}

At this time, you can request the data normally by accessing the html file just now.

Write as middleware

The above is only for a certain interface. If you want all interfaces to be cross-domain requests, you can write it as a middleware. The above is to allow cross-domain requests for all domains. In fact, there are still some problems. Here I will The address that allows cross-domain requests to be written to a map, originally wanted to write to an array or slice, but found that when judging whether it exists, there is no in operator in golang, and it is necessary to traverse the array or slice, and the complexity is O(n ) , or use map.

In addition, if the request method is OPTIONS, such as a websocket request, an OPTIONS request will be sent first. At this time, you can return directly without checking.

package midwares

import (
	"github.com/gin-gonic/gin"
)

func CheckCors() gin.HandlerFunc {
	//这里可以处理一些别的逻辑
	return func(c *gin.Context) {
		// 定义一个origin的map，只有在字典中的key才允许跨域请求
		var allowOrigins = map[string]struct{}{
			"http://127.0.0.1:8888": struct {}{},
			"https://www.yangyanxing.com": struct {}{},
		}
		origin := c.Request.Header.Get("Origin") //请求头部
    method := c.Request.Method
		if method == "OPTIONS"{
			c.AbortWithStatus(http.StatusNoContent)
		}
		if origin!=""{
			if _, ok:=allowOrigins[origin];ok{
				c.Header("Access-Control-Allow-Origin", origin)
				c.Header("Access-Control-Allow-Methods", "POST, GET, OPTIONS, PUT, DELETE, UPDATE")
				c.Header("Access-Control-Allow-Headers", "Origin, X-Requested-With, Content-Type, Accept, Authorization")
				c.Header("Access-Control-Expose-Headers", "Content-Length, Access-Control-Allow-Origin, Access-Control-Allow-Headers, Cache-Control, Content-Language, Content-Type")
				c.Header("Access-Control-Allow-Credentials", "true")
			}
		}
		c.Next()
	}
}
