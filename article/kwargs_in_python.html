Double star parameter expansion problem in python

Publish: 2019-07-11 Classify at: Python
Words:884 Read:≈ 5min

Indefinite parameters are allowed in python functions. We are accustomed to this kind of parameters, and this way of writing is very convenient, but we are not very familiar with the operating mechanism behind it. This kind of double star indefinite parameter problem is also recorded here.

# coding:utf-8

def test(name, age, *args, **kwargs):
    print("args:",args)
    print("kwargs:",kwargs)
    print("name:", name)
    print("age", age)
    for i in args:
        print("args:", i)
    for k, v in kwargs.items():
        print("kwargs:", k, v)


if __name__ == '__main__':
    test('yang', 18, 'is', 'a', 'good', 'man', sex="male")

The test result of this script is:

args: ('is', 'a', 'good', 'man')
kwargs: {'sex': 'male'}
name: yang
age 18
args: is
args: a
args: good
args: man
kwargs: sex male

We see that the parameters for calling the function test are 'yang', 18, 'is', 'a', 'good', 'man', sex="male", python assigns yang and 18 to Name and age, assign 'is', 'a', 'good', 'man' to the args tuple, and convert sex="male" to a dictionary to kwargs.

What exactly does the double star operation do?

We know kwargs is a dictionary, but have you considered what **kwargs is? Print it out and see.

When I try to print **dict, I get an error, which means that name is not a parameter of the print function, so what parameters does the print function in the standard library have? Let’s take a look at the definition of print in the standard library print(self, *args, sep=' ', end='\n', file=None), It does not have a formal parameter of name. If you use print(**d), will it have the same effect as print(name=“yangyanxing”)?

They reported the same error, here we first assume that using **dict is actually to expand the items in a dictionary in turn, such as d={’name’:“yangyanxing”}, use **d to return name=“yangyanxing”.

# coding:utf-8

def test(name, age):
    print("name:", name)
    print("age", age)


if __name__ == '__main__':
    d = {"name":"Yang Yanxing",'age':18}
    test(**d)

Here the test function requires two parameters, name and age, but we did not pass these two parameters directly when passing the parameters, but passed a **dictionary, the ** will expand the dictionary into key= value and passed to the test function. So the following output is obtained

name: Yang Yanxing
age 18

What if the incoming dictionary contains not only the parameters required by the function, but also more parameters?

# coding:utf-8


def test(name, age,**kwargs):
    print("name:", name)
    print("age", age)
    print("kwargs:", kwargs)


if __name__ == '__main__':
    d = {"name":"Yang Yanxing",'age':18,
         'sex':"male",'school':"bjut"}
    test(**d)

The result it gets is

name: Yang Yanxing
age 18
kwargs: {'school': 'bjut', 'sex': 'male'}

As you can see, it puts the redundant key-value pairs in the dictionary into the kwargs dictionary.

# coding:utf-8


def test(name,age,*args,**kwargs):
    print("args:", args)
    print("kwargs:",kwargs)
    print("name:",name)
    print('age:',age)


if __name__ == '__main__':
    d = {"name1":"Yang Yanxing",'age1':18}
    l = ['a','b','c']
    t = ('d','e','f')
    test(*l,*t,**d)

Here we expand the list l and tuple t to the test function, which is equivalent to test('a','b','c','d','e','f',name1="Yanxing Yang ",age1=18), then get the following output

args: ('c', 'd', 'e', ​​'f')
kwargs: {'age1': 18, 'name1': 'Yang Yanxing'}
name: a
age: b

There are several issues to pay attention to here. Since **kwargs can only be defined at the end of function parameters, there can be no formal parameters after it, but the * parameter can not limit the position of parameters, so the above code is defined in When d, there can no longer be the definition of name and age, because when passing parameters, ‘a’ has been assigned to name, and ‘b’ has been assigned to age, so if there is a definition of name later, parameters will be reported Duplicate definition error. But it can be defined as

# coding:utf-8


def test(*args,name,age,**kwargs):
    print("args:", args)
    print("kwargs:",kwargs)
    print("name:",name)
    print('age:',age)


if __name__ == '__main__':
    d = {"name":"Yang Yanxing",'age':18,'sex':'male'}
    l = ['a','b','c']
    t = ('d','e','f')
    test(*l,*t,**d)

Because *args is placed as the first parameter, it is test('a','b','c','d','e','f',name="Yang" when calling the function Yan Xing", age=18, sex='male'), if there is no name and age in the definition of d, an error will be reported.

I think this way of writing is not very elegant, but I have seen some libraries have similar ways of writing, so I don’t need to be too confused when I see it later, just understand that ** and * are just the following dictionary and list or The tuple can be displayed.
