Several ways to execute commands in the go language

Publish: 2019-08-23 Classify at: Go
Words:1155 Read:≈ 6min

The commands used by the go language to execute all systems are still a bit complicated compared to python. Executing commands is a very common requirement, such as calling a system command, starting an exe, etc. There are several situations here, which will be summarized later.

  • only execute the command, do not output the result
  • Execute the command and get the output
  • Blocking and asynchronous execution

The following takes ping www.baidu.com as an example to execute various commands in sequence, mainly using os/exec in the standard library

When executing commands, we mainly use the Cmd structure method of the os/exec package owner. The Cmd structure is defined as follows Cmd structure definition

But we generally do not construct the Cmd structure directly, but return a Cmd structure pointer through the exec.Command() function For example, exec.Command(“ping”,“ www.baidu.com ”) ping is the command, and “ www.baidu.com ” is the parameter. After getting *Cmd, the structure method Run, Start and other methods are used to execute the actual execution. Order.

To execute a command, you can use the Run() or Start() method. Run is a blocking execution, and Start() is a non-blocking execution.

package main

import (
	"fmt"
	"os/exec"
)

func main() {
	command := exec.Command("ping","www.baidu.com")
	err := command.Run()
	if err != nil{
	fmt.Println(err.Error())
	}
}

The program outputs nothing, but exits after a pause. If you replace it with command.Start(), the program will stop immediately after running.

If you want to get the execution result of os.system like python, in fact, the result here should be ExitError, the exit code of the program, what should you do? There is a special structure ExitError in the exec package. You can use some of its methods to get the ExitCode, but if you want to get the ExitCode, you can get the end of the command, that is, the operation to be blocked. Use the Run() method above to block Wait for the execution result. After using the Start() method, you can also use the Wait() method to wait for the execution to end.

package main

import (
	"fmt"
	"os/exec"
	"syscall"
)

func main() {
	command := exec.Command("ping","www.baidu.com")
	err := command.Start()
	if err != nil{
		fmt.Println(err.Error())
	}
	if err = command.Wait();err!=nil{
		fmt.Println(err.Error())
	}else{
		fmt.Println(command.ProcessState.Pid())
		fmt.Println(command.ProcessState.Sys().(syscall.WaitStatus).ExitCode)
	}
}

package main

import (
"bytes"
"fmt"
"os/exec"
"syscall"
)

func main() {
	command := exec.Command("ping","www.baidu.com")
	outinfo := bytes.Buffer{}
	command.Stdout = &outinfo
	err := command.Start()
	if err != nil{
		fmt.Println(err.Error())
	}
	if err = command.Wait();err!=nil{
		fmt.Println(err.Error())
	}else{
		fmt.Println(command.ProcessState.Pid())
		fmt.Println(command.ProcessState.Sys().(syscall.WaitStatus).ExitCode)
		fmt.Println(outinfo.String())
	}
}

package main

import (
"bytes"
"fmt"
"os/exec"
"strings"
)

func main() {
	var outInfo bytes.Buffer
	cmd := exec.Command("nslookup")
	cmd.Stdin = strings.NewReader("set q=mx\npython.org\nexit\n")
	cmd.Stdout = &outInfo
	cmd.Run()
	fmt.Println(outInfo.String())
}

package main

import (
"bytes"
"fmt"
"os/exec"
)

func main() {
	cmd := exec.Command("copy","1.txt","2.txt")
	var output bytes.Buffer
	cmd.Stdout = &output
	e :=cmd.Run()
	if e != nil{
		fmt.Println("run error :" + e.Error())
	}else{
		fmt.Println(output.String())
	}
}
