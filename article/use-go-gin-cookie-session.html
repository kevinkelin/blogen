Use gin to build cookie and session of api background system

Publish: 2022-01-04 Classify at: Go
Words:948 Read:≈ 5min

The previous articles are all stateless requests. When dealing with stateful requests, such as user login scenarios, the web system usually uses cookies or sessions to record user status. This article records cookies and sessions under the gin framework. usage of.

read cookies

There is a Cookie method in gin.Context, which returns the cookie you want to get. If it does not exist, it will return an error http: named cookie not present, you can judge whether there is an error by judging whether there is this error related cookies.

func (ApiV1) Cookies(c *gin.Context) {
	cookie, err := c.Cookie("user")
	if err != nil{
		c.JSON(200, gin.H{"msg": err.Error()})
	}else{
		c.JSON(200, gin.H{"msg": cookie})
	}
}

set cookies

You can use the SetCookie method to set cookies, this method has many parameters

func (c *Context) SetCookie(name, value string, maxAge int, path, domain string, secure, httpOnly bool)

The meaning of these parameters is as follows

  • The first parameter name is the cookie name;
  • The second parameter value is the cookie value;
  • The third parameter maxAge is the validity period of the cookie. When the cookie exists for more than the set time, the cookie will be invalid, it is no longer our valid cookie, and its time unit is second;
  • The fourth parameter path is the directory where the cookie is located;
  • The fifth domain is the domain, which means the scope of our cookie, which can be localhost or your domain name, depending on your own situation;
  • The sixth secure indicates whether it can only be accessed through https, if it is true, it can only be https;
  • The seventh httpOnly indicates whether the cookie can be manipulated by js code. When it is true, it cannot be obtained by js

func (ApiV1) SetCookie(c *gin.Context) {
	cookie, err := c.Cookie("user")
	if err != nil{
		c.SetCookie("user", "yangyanxing",3600, "/", "127.0.0.1",false, false)
		c.JSON(200, gin.H{"msg": "cookie set successfully"})
	}else{
		c.JSON(200, gin.H{"msg": cookie})
	}
}

Operation session

Although the cookie is very convenient, its security still has some problems. The client can modify it arbitrarily. At this time, the session can also be used to store the state. The gin framework itself does not support sessions, but we can use a third-party library https://github.com/gin-contrib/sessions to operate sessions, this session framework can store information in different places

Storage engines supported by the gin-contrib/sessions middleware:

  • cookies
  • memstore
  • redis
  • memcached
  • mongodb

package main

import (
	"github.com/gin-contrib/sessions"
	"github.com/gin-contrib/sessions/cookie"
	"github.com/gin-gonic/gin"
)

func main() {
	r := gin.Default()
	// Create a cookie-based storage engine, the yangyanxing parameter is the key used for encryption
	store := cookie.NewStore([]byte("yangyanxing"))
	// Set the session middleware, the parameter mysession refers to the name of the session and the name of the cookie
	// store is the storage engine created earlier, we can replace it with other storage engines
	r.Use(sessions.Sessions("mysession", store))
	apiv1_h := handlers.ApiV1{}
	apiv1 := r.Group("/api/v1")
	{
		apiv1.GET("/session/get", apiv1_h.GetSession)
		apiv1.GET("/session/set", apiv1_h.SetSession)

	}
	r.Run(":8080")
}

store := cookie.NewStore([]byte("yangyanxing")) This line of code initializes a storage engine with a key for encryption and decryption

r.Use(sessions.Sessions("mysession", store)) This is a middleware, the cookie is stored with the name mysesson, and the store is the storage engine initialized above.

func (ApiV1) GetSession(c *gin.Context) {
	session := sessions.Default(c)
	if value:= session.Get("user"); value==nil{
		c.String(200, "There is no user in the session")
	}else{
		c.String(200, value.(string))
	}
}

func (receiver ApiV1) SetSession(c *gin.Context) {
	session := sessions.Default(c)
	session.Set("user", "yangyanxing")
	err := session.Save()
	if err != nil {
		c.String(200, err.Error())
	}
	c.String(200, "Set the session successfully")
}

Note that the session.Save() method must be called to write operations on the session, such as setting, modifying, deleting, etc., otherwise it will not take effect.

func (ApiV1) ClearSession(c *gin.Context) {
	session := sessions.Default(c)
	//session.Delete("user")
	session.Clear()
	err := session.Save()
	if err != nil {
		c.String(200, err.Error())
		return
	}
	c.String(200, "Cleaning succeeded")
}

Delete is to delete a single value, Clear is to clear all, but note that even if Clear is used, it only clears the information in the session, but the mysession in the cookie still exists, but the value is different.

Use redis to store session

The usage is the same as the cookie, but it is different when initializing the storage engine

store, _ := redis.NewStore(10, "tcp", "localhost:6379", "", []**byte**("secret"))

Parameter Description

  • The first parameter - the maximum number of idle connections in redis
  • a parameter - the number of communication protocols tcp or udp
  • Parameters - redis address, format, host:port
  • 4th parameter - redis password
  • 5th parameter - session encryption key

Notice:

The store initialized with redis.NewStore uses redis db0 by default. More often, we will use redis.NewRediStoreWithDB to specify the db
