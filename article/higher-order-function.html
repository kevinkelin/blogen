The use of map, lambda, higher-order functions and decorators in python

Publish: 2018-07-01 Classify at: Python
Words:1223 Read:≈ 6min

When friends with a little python foundation see some functions written by someone, they will intentionally write some higher-order functions, either with a map, or a lambda, and a higher @ decoration There is nothing difficult about the other things. The usage is more advanced. If you can follow its logic and use it in practical applications accurately, it will still bring a certain improvement in the efficiency of the program. As for readability, when When you are accustomed to these usages, you will feel that its readability is also very good. The reason why you are not used to it is mainly because you do not understand its usage. When it comes to readability, this is another topic, which will be discussed in detail later. Let’s talk about the use of higher-order functions and decorators today. I hope that friends who read this article have benefited from it.

Higher order functions

Definition of higher-order functions: A function that can receive another function as a parameter is called a higher-order function. The parameter of a regular function is usually a variable or an object, such as defining a summation function

    return x + y

if __name__ == '__main__':
    print sum(3,4)

The parameters passed in here, x and y are both integers, and the return is their sum. Can a variable be a function?

Assign the sum() function to the variable s, s is now a function, and the name of s is ‘sum’, Calling s(6,7) returns sum(6,7).

The above is very simple. Let’s take a look at the definition of higher-order functions. A function can receive another function as a parameter. This function is called a higher-order function, Since a function can also be used as a parameter of another function, let’s define another function

The return value of the sum3 function is sum4, but when printing f3 directly, it displays <function sum4 at 0x02142DB0>. To execute this function, you need to call and execute an f3 function f3()

map function

The above code does not seem to have any specific application. As long as it is for illustration, the function can also be passed as a parameter to another function. Let’s take a look at the map function. The map() function receives two parameters, one is a function and the other is an Iterable. The map applies the incoming function to each element of the sequence in turn, and returns the result as a new Iterator. The simplest application, given a list, returns the square of each element in the list.

Called a function, in fact, it is a function without a name, one-time use, suitable for that kind of simple expression without defining a function, the above sum and cube.

Note that when lambda is used in the map function, if its parameters need two or more, then a list of the corresponding number should also be passed in the map parameter, and its calculation is the expression of the corresponding position in the list, For example, for summation, it is to sum the first element in list 1 and the first element in the list into the first recommendation in the returned list. The length of the list must be the same, and the first list cannot have 4 elements. The second list has 3 or 5, and the second list can only be 4.

decorator

With some of the above foundations, let’s take a look at the focus of this article - the decorator First look at the definition of a simple function to print the current date and time

from datetime import datetime

def showTime():
    print datetime.now()


showTime()

Now, suppose we want to enhance the functionality of the showTime function, for example, to automatically print the log before and after the function call, but do not want to modify the definition of the showTime function. This way of dynamically adding functionality during code execution is called “Decorator”.

define a function

1
2
3
4
5

def log(func):
    def wrapper(*args, **kw):
        print 'call %s():' % func.__name__
        return func(*args, **kw)
    return wrapper

Observe this log function, its parameter is a function, when you want to execute the function passed in the parameter, wrap it in a layer, do some other things, and then return the function, return wrapper So essentially this log It is a higher-order function that returns a function. Refer to the definition of the sum2 function above, and then look at the function described above as the return value. In fact, it has executed the func function here. For the above code, it is print datetime.now() The use of the decorator is to add the @ sign to the definition of the function

def log(func):
    def wrapper(*args, **kw):
        print 'call %s():' % func.__name__
        return func(*args, **kw)
    return wrapper

@log
def showTime(a,b):
    print a
    print datetime.now()
    print b

showTime('yang','fan')

After adding the decorator, let’s break down the call to the showTime function, which actually executes log(showTime("yang","fan"))

If the decorator itself needs to pass in parameters, then you need to write a higher-order function that returns the decorator, which will be more complicated to write. For example, to customize the text of the log:

def log2(text):
    def decorator(func):
        def wrapper(*args, **kw):
            print('%s %s():' % (text, func.__name__))
            return func(*args, **kw)
        return wrapper
    return decorator
    
@log2("yyx")
def showTime(a,b):
    print a
    print datetime.now()
    print b

showTime('yang','fan')

The execution of the above is like this: First execute log2('yyx') to return the decorator function, then call the returned function (decorator), the parameter is the showTime function, the return value is finally the wrapper function, and the wrapper function will execute the showTime function, so after the The call after log2 decoration should be broken down to log2(“yyx”)(showTime(“yang”,“fan”)).

Therefore, when writing a decorator, it is necessary to distinguish whether the decorator itself needs to have parameters. If not, it can be wrapped in one layer. If there are parameters, it needs to be wrapped in two layers.
