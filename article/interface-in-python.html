In python, how to achieve the effect of abstract class?

Publish: 2022-07-14 Classify at: python
Words:636 Read:≈ 3min

We all know that there is the concept of interface in java. A class can implement multiple interfaces, but it must implement all the methods in the interface, but there is no concept of interface in python. Classes can inherit through multiple inheritance Since there are multiple classes, now if we want to implement the effect of the interface in java, it is mandatory that the subclass must implement some methods, how to do it?

Implementation of abstract class

Python has a ABCMeta class. If the metaclass is set to ABCMeta when the class is defined, and an abstract method is defined, the class cannot be initialized directly.

import abc


class Runable(metaclass=abc.ABCMeta):

    @abc.abstractmethod
    def run(self):
        pass


r = Runable()

Above we defined a Runable class and set the metaclass to abc.ABCMeta, we defined a run method in this class and decorated it with @abc.abstractmethod, in this case, when trying to use r = Runable() to initialize an instance, an exception will be thrown TypeError: Can't instantiate abstract class Runable with abstract method run

But if the @abc.abstractmethod decorator is removed, it can be instantiated normally

We can also define classes by inheriting from the abc.ABC class, the effect is the same as above, the abc.ABC class is a wrapper class

    """Helper class that provides a standard way to create an ABC using
    inheritance.

    __slots__ = ()

class Runable(abc.ABC):

    def run(self):
        print("can run.....")

If you neither inherit abc.ABC nor set metaclass=ABCMeta when defining the class, even if you add the abstractmethod decorator to the method, it will not play the role of an abstract class, or you can Normal initialization instance. But if no abstract method is defined, it can also be a normal instance

So we come to the following conclusion, to achieve the effect of abstract class, that is, to meet two conditions

  1. The class inherits from abc.ABC or sets the metaclass to abc.ABCMeta
  2. Define an abstract method in a class, that is, a method decorated with abc.abstractmethod

Subclasses that implement abstract classes

The original intention of an abstract class is to define rules that require subclasses to implement which methods. When a subclass inherits from an abstract class and implements all the methods in the abstract class, the class can be instantiated normally, otherwise it will not. can be instantiated

class Runable(metaclass=abc.ABCMeta):

    @abc.abstractmethod
    def run(self):
        pass

    @abc.abstractmethod
    def walk(self):
        pass

class Dog(Runable):
    def run(self):
        print("dog run ")

The above Dog class only implements the run method of Runable, and does not implement the walk method, so it still cannot be instantiated normally.

There is a point to note here. The subclass does not necessarily require the same function signature as the abstract method for the abstract method in the parent class. This is really surprising. üòÇ see the following code

class Runable(metaclass=abc.ABCMeta):

    @abc.abstractmethod
    def run(self):
        pass

    @abc.abstractmethod
    def walk(self, name):
        pass

class Dog(Runable):
    def run(self):
        print("dog run ")

    def walk(self):
        print("dog walk")
        
d = Dog()
d.run()
d.walk()

The signature of the walk method defined in the abstract class Runable is def walk(self, name): There needs to be a name parameter, but the subclass does not have this parameter when it is implemented. In python, it is considered to be implemented. the walk abstract method, but the IDE will give a warning Signature of method 'Dog.walk()' does not match signature of base method in class 'Runable' The above code can run normally, that is to say, the Dog class can be initialized normally, so don’t complain about this!
