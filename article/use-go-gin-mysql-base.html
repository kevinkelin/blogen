Most of the articles and tutorials are based on GORM, but I am not really interested in ORM. This article will perform CURD operations in native sql language.

package db

import (
	"database/sql"
	"fmt"
	_ "github.com/go-sql-driver/mysql"
	"time"
)

func InitDB() *sql.DB {
	dsn := fmt.Sprintf("%s:%s@(%s)/%s?charset=%s&parseTime=true&loc=Local",
	"root", "123456", "127.0.0.1:3306", "gintest", "utf8")

	if conn, err := sql.Open("mysql", dsn); err != nil {
		panic(err.Error())
	} else {
		conn.SetConnMaxLifetime(7 * time.Second) //Set the idle time, which is shorter than when mysql actively disconnects
		conn.SetMaxOpenConns(10)
		conn.SetMaxIdleConns(10)
		return conn
	}
}

The sql.Open method starts to connect to the database. The first parameter is the type of database to be connected. The second parameter is dsn, which can be understood as the parameter information for connecting to the database.

After the connection, three settings are made to the connection.

conn.SetConnMaxLifetime is to set the idle time of the connection, why to set this parameter and how much to set is particular.

Some MySQL servers will automatically disconnect idle connections for performance reasons, the default is 8 hours, but the general dba will not be set that long, it is likely to be set to 10 seconds or less, so this parameter should be set more. Short, this parameter can be viewed by logging in to the MySQL server and executing show global variables like '%timeout%';, there is a wait_timeout value, and the value here should be set shorter than this value.

The two methods conn.SetMaxOpenConns(10) and conn.SetMaxOpenConns(10) will initialize a connection pool to ensure that there will be enough connections in the pool. How much this value should be set depends on the concurrency of the application.

But the value of SetMaxIdleConns should not be less than the value set by SetMaxOpenConns, otherwise it will cause frequent connection creation.

The official explanation is

db.SetMaxIdleConns() is recommended to be set same to db.SetMaxOpenConns(). When it is smaller than SetMaxOpenConns(), connections can be opened and closed much more frequently than you expect

Another thing to note here is that when importing, there is a _ "github.com/go-sql-driver/mysql" line of code that cannot be missing. This code will execute the init method for a while to initialize some variables. Without this line, it will panic because some variables are not initialized.

package main

import (
	"github.com/gin-gonic/gin"
	"yyxtest/db"
	"yyxtest/global"
	"yyxtest/handlers"
)

func main() {
	r := gin.Default()
	global.Mysql = db.InitDB()
	userv1_h := handlers.UserV1{}
	userv1 := r.Group("/user/v1")
	{
		usv1.GET("/check", usv1_h.CheckUsers)
	}
	r.Run(":8080")
}

global.Mysql = db.InitDB() This line of code initializes the global Mysql object, then we create a dao folder, and then create a userdao.go file, which encapsulates the database query. First write a query for all records. function.

package dao

import "yyxtest/global"

type user struct {
	ID int `json:"id"`
	Name string `json:"name"`
	Age int `json:"age"`
}


func GetAllUsers() []user {
	rows, err :=global.Mysql.Query("select * from user_infos")
	if err != nil{
		return nil
	}
	var persons = make([]user, 0)
	for rows.Next() {
		var a user
		err := rows.Scan(&a.ID, &a.Name, &a.Age)
		if err != nil {
			return nil
		}
		persons = append(persons, a)
	}
	return persons
}

The GetAllUsers function executes select * from user_infos, and returns the query result to the structure of []user, and then writes the routing processing class, usv1_h.CheckUsers

package handlers

import (
	"github.com/gin-gonic/gin"
	"yyxtest/dao"
)

type UserV1 struct {}

func (UserV1) CheckUsers(c *gin.Context) {
	users := dao.GetAllUsers()
	if users == nil{
		c.JSON(200, gin.H{"error": 1, "msg": "Query failed"})
		return
	}
	c.JSON(200, gin.H{"error":0, "msg":"query success", "users": users})
}

SQL injection problem

If the user’s input is not verified when querying the database, but the string concatenation is completely performed, it will cause the problem of SQL injection, such as the following code

func GetUser(name string) (user, error) {
	sqlstr := fmt.Sprintf("select * from user_infos where `name`='%s'", name)
	row := global.Mysql.QueryRow(sqlstr)
	var u user
	err := row.Scan(&u.ID, &u.Name, &u.Age)
	if err != nil {
		return user{}, err
	}
	return u, nil
}

String concatenation directly in sqlstr will cause injection problems.

func InsertUser(name string, age int) error {
	exec, err := global.Mysql.Exec("INSERT into user_infos(name, age) values ​​(?,?)", name, age)
	if err != nil {
		return err
	}
	_, err = exec.LastInsertId()
	if err != nil {
		return err
	}
	return nil
}

func (UserV1) AddUser(c *gin.Context) {
	name := c.Query("name")
	age := c.Query("age")
	age_i, err:=strconv.Atoi(age)
	if err!=nil{
		c.JSON(200, gin.H{"error": 1, "msg": "age parameter is incorrect"})
	}else{
		err = dao.InsertUser(name, age_i)
		if err!= nil{
			c.JSON(200, gin.H{"error": 2, "msg": err.Error()})
		}else{
			c.JSON(200, gin.H{"error": 1, "msg": "Insert successfully"})
		}
	}

}
