Four, classic class and new class

When talking about multiple inheritance above, I mentioned that there are two problems that will be understood after reading the classic classes and new-style classes. I will explain in detail here. Don’t be fooled by the name. There is nothing special, that is, in python2, When a class does not inherit the task class, the above Person class is used. It is a classic class. If a class inherits from “object”, it is a new-style class. In python3, it is a new-style class by default, that is, both It is inherited from object. That is to say, on the classic class (old-fashioned class or called the old class), the properties or methods of the search class are depth-first, that is, search from the bottom to the top. In the classic class, when calling fun.save() , first search from the class D, and find that it does not have a save method, then search from the B class it inherits, and neither does it in B, then search from B’s parent class A, which has a save method, Then stop the search and call the save method of class A. Its search order is D, B, C, A. Since class C has a save method, the save method of class C is called directly to stop the search.

        self.sex = sex
        self.name = name

    def showInfo(self):
        print(self.name)

    def showInfo2(self):
        print('parent fun')


class Student(Persion):
    def __init__(self, name, age, grade, sex='M'):
        Persion.__init__(self,name,age,sex)
        self.grade = grade

    def showInfo2(self):
        super(Student, self).showInfo2()
        super(Student, self).showInfo()
        print('child fun')


s = Student("yang",32,'Junior')
s.showInfo2()

parent fun
yang
child fun

Note that if you want to use the super class, it generally has two parameters, the first parameter is a class name, the second parameter is an instance (or a class), if the second parameter is an instance (such as self here) , you can understand that sefl is an instance) needs to satisfy isinstance(obj, type) == True, if the second parameter is also a class name, then it needs to satisfy issubclass(type2, type) == True, That is, if the second class is a subclass of the first class, what does super do later? super contains two very important pieces of information: an MRO (Method Resolution Order) list and a class in the MRO. It was introduced above that the new-style class has a mro() method. Calling the mro method of a class will display the search order of methods and properties, such as Calling D.mro() shows

1

[<class '__main__.D'>, <class '__main__.B'>, <class '__main__.C'>, <class '__main__.A'>, <class 'object'>]

When a class is initialized with super, the corresponding method will be searched in the following classes. For example, the mro list of a certain class is [A, B, C, D, E, object], when the super class is initialized with super(C,self), this mro will only be searched from the classes after C, such as calling super(C,self) .test() method, it will only find out whether there is a test() method from the D, E, object classes, which also implements the method of calling the parent class. see the code below
