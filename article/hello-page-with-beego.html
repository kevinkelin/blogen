Use beego to build the project homepage

Publish: 2021-05-29 Classify at: Go
Words:807 Read:≈ 4min

I recently tried to use beego to initialize a project, and follow the documentation step by step. Since the first attempt to use golang to build Here I also record the problems and solutions I encountered during initialization. Since I am also a novice, the problems encountered may be relatively low-level, but this is also used as some basis for the future use of the framework.

1. Install beego and bee tools

There are two ways to install the tool, one is to use the go get command, and the other is to use the go mod command, but both methods have some pre-operations

  1. Device goproxy, speed up domestic download speed
  2. Add $GOPATH/bin to the environment variable

setup goproxy

Due to the domestic network environment, you need to set up a proxy to quickly download the third-party package of golang.

At present, goproxy.io and goproxy.cn are widely used. goproxy.cn is built by Qiniuyun. The setting method is the same, but the path is different.

My local go version is 1.16, and those below 13 should be upgraded. After all, we still see that go mod should be the mainstream in the future.

1
2

$ go env -w GO111MODULE=on
$ go env -w GOPROXY=https://goproxy.cn,direct

Add $GOPATH/bin to the environment variable PATH.

The second step is not necessary, because you want to use the bee tool to quickly generate the base directory of the beego website, so if you want to use the bee tool in any directory, you need to add the bin directory to the environment variable PATH.

1
2

export GOPATH=/Users/yangyanxing/go
export PATH=$PATH:$GOPATH/bin

The above command is a temporary change. If you want to make a permanent change, you need to modify the following files and add the above code to any of the files.

1
2
3
4
5
6
7

a. /etc/profile
b. /etc/paths
c. ~/.bash_profile
d. ~/.bash_login
e. ~/.profile
f. ~/.bashrc
e. ~/.zshrc

a and b are system level, it is recommended not to modify, if you use bash, modify ~./bashrc, if you use zsh, modify ~/.zshrc

1.1 Use go get command to install beego and bee tools

install beego

1

go get github.com/beego/beego/v2

Install the bee tool, the bee tool can quickly initialize a beego project, also run the beego project, and can be hot loaded

1

go get -u github.com/beego/bee/v2

Question 1, where are these packages installed?

View the GOPATH directory of the system through go env GOPATH, the installed third-party packages are downloaded to the $GOPATH/pkg/mod directory, and the directory structure is also based on the path structure of the package.

1.2 Use go mod download to download third-party packages

You can also use the go mod download command to download third-party packages. Generally, after cloning other people’s projects to the local, the local first downloads the corresponding library version according to the package information in the go.mod file, which is somewhat similar to the one in python. The role of the requirements.txt file is to create a virtual environment in python, but there is no concept of virtual environment in go. All third-party packages are put together and will be distinguished according to the version number.

First use the go mod init modtest command to initialize the mod file in a blank directory, and a go.mod file will be generated here

See its contents cat go.mod

1
2
3

module modtest

go 1.16

Here you can add dependencies for third-party packages

1
2
3
4
5
6

module beetest

go 1.16

require github.com/beego/beego/v2 v2.0.1
require github.com/smartystreets/goconvey v1.6.4

At this time, in the go.mod directory, use the go mod download command to download the third-party package.

No matter using go get or go mod download command, the download progress bar will not be displayed, which looks awkward.

Second, use the bee tool to generate the beego project structure

We use a new directory to generate the beego project, using bee new beetest will generate a new directory beetest, here will generate a new go.mod file

Go to the beetest directory

1
2
3

bee new beetest
cd beetest
go mod download

Regardless of the directory structure in the project, use the bee run command to start the service. Here, open http://127.0.0.1:8080 in the browser to see the beego welcome page.

Question 2, if you don’t use go mod download, you can’t use it, why?

Because after initializing the project, there is only go.mod file, no go.sum file, this file will record some library information, if not, it will prompt

1
2
3
4

go: github.com/beego/beego/v2@v2.0.1: missing go.sum entry; to add it:
go mod download github.com/beego/beego/v2
2021/05/29 17:05:34 ERROR ▶ 0004 Failed to build the application: go: github.com/beego/beego/v2@v2.0.1: missing go.sum entry; to add it:
go mod download github.com/beego/beego/v2

So you need to use the go mod download command to download the package and write the information to the go.sum file.
